# 저장소 작업 지침

## 기준 문서

- `2027_전문대_입시상담앱_실행개발문서_v2.md`를 상위 실행 명세로 사용한다.
- `CODEX_MASTER_PROMPT_2027_전문대_입시상담앱_v3.md`의 실행 순서와 단계 게이트를 따른다.
- 두 원본 문서는 삭제, 이동, 축약 또는 덮어쓰지 않는다.

## 변경 원칙

- 현재 Phase의 작업 카드에 명시된 경로만 수정한다.
- 사용자 변경과 원본 자료를 보존하고 파괴적 Git 명령을 사용하지 않는다.
- Flask Jinja2 SSR, HTML, Tailwind CSS를 기본으로 하고 필요한 상호작용만 Vanilla JavaScript와 fetch API로 구현한다.
- SPA 프레임워크와 TypeScript를 사용하지 않으며 핵심 기능은 JavaScript 없이 동작해야 한다.
- 학생용·교사용 화면은 별도 A4 ` print` CSS를 유지한다.
- PostgreSQL을 기본 DB로 사용하고 애플리케이션 컨테이너와 DB 컨테이너, DB 저장 volume을 분리한다.
- 지원자격 판정 전 성적 계산을 수행하지 않는다.
- 공식 근거가 없는 규칙을 추정하거나 공개하지 않는다.

## 개인정보와 데이터

- 실제 학생 자료, 원본 PDF·엑셀, 업로드 파일, OCR 파생물, SQLite DB, `.env`, API 키를 커밋하지 않는다.
- 테스트에는 합성 익명 자료만 사용한다.
- Git에 둘 수 있는 전형 데이터는 검수 완료 후 공개 가능한 `data/seed/` 자료뿐이다.
- 커밋 전 `python scripts/check_sensitive_data.py`를 실행한다.

## Git

- 커밋 전에 `git status`와 `git diff`를 검토한다.
- 한 커밋은 하나의 Phase 또는 하나의 독립 작업 범위만 포함한다.
- 모든 커밋 메시지는 자연스러운 한국어로 작성한다.

## 테스트 원칙
- Playwright E2E 테스트는 HTTP 상태 코드만 확인하지 않고, 사용자 행동 후 핵심 화면 요소·저장 데이터·오류 없는 동작을 검증한다.

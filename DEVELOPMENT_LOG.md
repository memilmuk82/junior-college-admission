# 개발 기록

## 2026-07-10 · Phase 0

- 빈 `main` 브랜치와 기존 요구사항 문서 두 개를 확인했다.
- 문서의 상하 관계와 Phase 경계를 검토했다.
- 저장소에 실제 학생 자료, PDF·엑셀, 업로드 파일, DB 또는 파생 이미지가 없음을 확인했다.
- 저장소 안전 정책, 초기 Flask 앱 셸, 검증 스크립트와 테스트를 추가했다.

## 2026-07-11 · Phase 1

- DB와 DB 통합 테스트를 PostgreSQL 17로 통일했다.
- 문서·규칙·학생성적 23개 핵심 테이블과 Alembic 최초 migration을 추가했다.
- SQLAlchemy 2.x 모델을 Flask-SQLAlchemy와 연결하고 Flask-Migrate CLI를 등록했다.
- 혼합연도 문서와 승인·출처·골든 테스트가 없는 공개 규칙을 DB 제약으로 차단했다.
- 원적교·위탁기관 성적 출처를 분리하고 임시 원본·파생물 삭제 검증 서비스를 추가했다.
- 단위 테스트 7건과 PostgreSQL 통합 테스트 7건이 통과했다.
- Ruff, mypy, 규칙 검증, 민감자료 검사가 통과했다.
- 실제 학생 자료와 실제 대학 규칙은 사용하지 않았다.

## 2026-07-11 · Phase 2 시작

- 입력 게이트웨이 작업 카드를 만들고 구조화 입력 정규화 계약부터 재개했다.
- CSV·XLSX·표 붙여넣기를 먼저 안정화한 뒤 OCR을 추가하는 단계 순서를 유지한다.
- XLSX 복수 시트와 앞부분 설명 행을 탐색해 표준 머리글 이후의 합성 행만 정규화한다.
- XLSX 입력 크기·시트 수·시트별 행 수 상한을 두고 외부 링크를 읽지 않는다.
- CSV·XLSX 학급표의 학생 식별자는 메모리 비교에만 사용하고 대상 학생 행과 과목 통계만 미리보기에 남긴다.
- 교사가 선택한 행만 `ImportBatch`와 학생 성적 테이블에 저장하고 원본·파생물 삭제 검증 뒤 transaction을 확정한다.
- 삭제 실패 시 DB 변경을 rollback하며 `P`는 숫자 0이 아닌 `raw_score_label`로 보존한다.
- 텍스트 PDF에서 고정 페이지를 가정하지 않고 교과학습발달상황 표를 탐색하며 페이지 출처를 유지한다.
- 세부능력 및 특기사항 이후 원문은 입력에서 제외하고 암호화·무텍스트 PDF는 임의 처리하지 않는다.
- PNG·JPEG·클립보드 이미지는 signature와 해상도를 검증한 뒤 컨테이너 내부 Tesseract 한국어·영어 OCR로만 처리한다.
- OCR 원문은 반환·로그·DB에 저장하지 않고 모든 OCR 행에 교사 검수 필요 이슈를 부여한다.
- 이미지형 PDF는 PDFium으로 페이지별 메모리 렌더링 후 로컬 OCR하며 경미한 EOF 손상은 렌더러가 복구할 수 있을 때만 처리한다.
- 한 페이지라도 렌더링·OCR이 실패하면 부분 결과를 반환하지 않고, 확인 행에는 원본 PDF 페이지 번호를 유지한다.
- 실제 학생 자료 없이 합성 익명 행만 테스트한다.
- 임시 검수 상태를 무작위 세션 하위에 저장하고 교사에게 원본 파일명이나 학생 식별자를 표시하지 않는다.
- 교사는 SSR 화면에서 표준 행을 수정·선택하며 JavaScript 없이도 선택 행만 확정할 수 있다.
- 검수·완료 응답은 브라우저 캐시를 금지하고 CSRF·CSP·referrer·frame 정책을 적용한다.
- 데스크톱·모바일·JavaScript 비활성 Playwright 흐름과 PostgreSQL 저장·폐기를 검증해 Phase 2를 완료했다.

## 2026-07-12 · Phase 3 시작

- 학생에게 하나의 전형 유형을 배정하지 않고 전형별 규칙에 사실을 독립 입력하는 모델을 추가했다.
- 학교 유형·졸업·직업위탁 상태와 기간·전학·검정고시 및 제한된 추가 사실을 표현한다.
- `all`·`any`·`not`과 비교 연산만 허용하고 문자열 수식, 임의 필드, 과도한 깊이·노드 수를 거부한다.
- 누락 사실은 3값 논리로 평가해 결격이 아니라 `INSUFFICIENT_DATA`로 분리한다.
- 게시·근거·독립 검증·골든 테스트·사람 승인 조건이 모두 있는 규칙만 실행한다.
- 설명 trace에는 학생의 실제 사실값을 복사하지 않고 규칙 ID·버전·조건 결과·누락 필드만 남긴다.
- 자격 상태가 `ELIGIBLE` 또는 `CONDITIONALLY_ELIGIBLE`이 아니면 계산 진입을 차단한다.
- 실제 대학 규칙이나 실제 학생 자료 없이 합성 사실·합성 규칙만 테스트했다.


## 확정 기술 결정

- uv로 Python 환경과 잠금 파일을 관리한다.
- PostgreSQL을 기본 DB로 사용하고 `web`·`db`·`postgres_data`를 분리한다.
- Jinja2 SSR을 기본으로 하며 SPA·TypeScript를 사용하지 않는다.
- 핵심 흐름은 JavaScript 없이 동작하고 별도 A4 인쇄 CSS를 사용한다.

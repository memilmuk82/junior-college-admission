# 개발 기록

## 2026-07-10 · Phase 0

- 빈 `main` 브랜치와 기존 요구사항 문서 두 개를 확인했다.
- 문서의 상하 관계와 Phase 경계를 검토했다.
- 저장소에 실제 학생 자료, PDF·엑셀, 업로드 파일, DB 또는 파생 이미지가 없음을 확인했다.
- 저장소 안전 정책, 초기 Flask 앱 셸, 검증 스크립트와 테스트를 추가했다.

## 2026-07-11 · Phase 1

- DB와 DB 통합 테스트를 PostgreSQL 17로 통일했다.
- 문서·규칙·학생성적 23개 핵심 테이블과 Alembic 최초 migration을 추가했다.
- SQLAlchemy 2.x 모델을 Flask-SQLAlchemy와 연결하고 Flask-Migrate CLI를 등록했다.
- 혼합연도 문서와 승인·출처·골든 테스트가 없는 공개 규칙을 DB 제약으로 차단했다.
- 원적교·위탁기관 성적 출처를 분리하고 임시 원본·파생물 삭제 검증 서비스를 추가했다.
- 단위 테스트 7건과 PostgreSQL 통합 테스트 7건이 통과했다.
- Ruff, mypy, 규칙 검증, 민감자료 검사가 통과했다.
- 실제 학생 자료와 실제 대학 규칙은 사용하지 않았다.

## 2026-07-11 · Phase 2 시작

- 입력 게이트웨이 작업 카드를 만들고 구조화 입력 정규화 계약부터 재개했다.
- CSV·XLSX·표 붙여넣기를 먼저 안정화한 뒤 OCR을 추가하는 단계 순서를 유지한다.
- XLSX 복수 시트와 앞부분 설명 행을 탐색해 표준 머리글 이후의 합성 행만 정규화한다.
- XLSX 입력 크기·시트 수·시트별 행 수 상한을 두고 외부 링크를 읽지 않는다.
- CSV·XLSX 학급표의 학생 식별자는 메모리 비교에만 사용하고 대상 학생 행과 과목 통계만 미리보기에 남긴다.
- 교사가 선택한 행만 `ImportBatch`와 학생 성적 테이블에 저장하고 원본·파생물 삭제 검증 뒤 transaction을 확정한다.
- 삭제 실패 시 DB 변경을 rollback하며 `P`는 숫자 0이 아닌 `raw_score_label`로 보존한다.
- 텍스트 PDF에서 고정 페이지를 가정하지 않고 교과학습발달상황 표를 탐색하며 페이지 출처를 유지한다.
- 세부능력 및 특기사항 이후 원문은 입력에서 제외하고 암호화·무텍스트 PDF는 임의 처리하지 않는다.
- PNG·JPEG·클립보드 이미지는 signature와 해상도를 검증한 뒤 컨테이너 내부 Tesseract 한국어·영어 OCR로만 처리한다.
- OCR 원문은 반환·로그·DB에 저장하지 않고 모든 OCR 행에 교사 검수 필요 이슈를 부여한다.
- 이미지형 PDF는 PDFium으로 페이지별 메모리 렌더링 후 로컬 OCR하며 경미한 EOF 손상은 렌더러가 복구할 수 있을 때만 처리한다.
- 한 페이지라도 렌더링·OCR이 실패하면 부분 결과를 반환하지 않고, 확인 행에는 원본 PDF 페이지 번호를 유지한다.
- 실제 학생 자료 없이 합성 익명 행만 테스트한다.
- 임시 검수 상태를 무작위 세션 하위에 저장하고 교사에게 원본 파일명이나 학생 식별자를 표시하지 않는다.
- 교사는 SSR 화면에서 표준 행을 수정·선택하며 JavaScript 없이도 선택 행만 확정할 수 있다.
- 검수·완료 응답은 브라우저 캐시를 금지하고 CSRF·CSP·referrer·frame 정책을 적용한다.
- 데스크톱·모바일·JavaScript 비활성 Playwright 흐름과 PostgreSQL 저장·폐기를 검증해 Phase 2를 완료했다.

## 2026-07-12 · Phase 3 시작

- 학생에게 하나의 전형 유형을 배정하지 않고 전형별 규칙에 사실을 독립 입력하는 모델을 추가했다.
- 학교 유형·졸업·직업위탁 상태와 기간·전학·검정고시 및 제한된 추가 사실을 표현한다.
- `all`·`any`·`not`과 비교 연산만 허용하고 문자열 수식, 임의 필드, 과도한 깊이·노드 수를 거부한다.
- 누락 사실은 3값 논리로 평가해 결격이 아니라 `INSUFFICIENT_DATA`로 분리한다.
- 게시·근거·독립 검증·골든 테스트·사람 승인 조건이 모두 있는 규칙만 실행한다.
- 설명 trace에는 학생의 실제 사실값을 복사하지 않고 규칙 ID·버전·조건 결과·누락 필드만 남긴다.
- 자격 상태가 `ELIGIBLE` 또는 `CONDITIONALLY_ELIGIBLE`이 아니면 계산 진입을 차단한다.
- 실제 대학 규칙이나 실제 학생 자료 없이 합성 사실·합성 규칙만 테스트했다.
- 지원자격·복수지원·결격 규칙을 기존 PostgreSQL 테이블에서 별도로 조회하고 서로 다른 결과 상태로 반환한다.
- 같은 전형의 활성 게시 규칙은 유형별 하나만 허용하는 PostgreSQL partial unique index를 추가했다.
- 규칙이 없거나 둘 이상이거나 payload가 제한형 계약을 벗어나면 명시적으로 실패한다.
- 복수지원은 대학별 총 횟수·캠퍼스별 횟수·금지 전형 조합을 데이터로 평가하며 불완전 이력은 `NEEDS_REVIEW`로 둔다.
- 민감 결격 사실은 bool 세션 입력으로만 받고 실제 값은 DB·규칙 payload·trace에 복사하지 않는다.
- 게시 규칙 seed 검사에서 지원자격·복수지원·결격 payload 계약도 검증해 Phase 3를 완료했다.

## 2026-07-12 · Phase 4 시작

- 지원자격이 `ELIGIBLE` 또는 `CONDITIONALLY_ELIGIBLE`인 경우에만 성적 범위 조회를 시작한다.
- `HOME_ONLY`, `VOCATIONAL_INCLUDED`, `VOCATIONAL_ONLY`, `EXCLUDE_VOCATIONAL_SEMESTER`를 출처·위탁학기 사실로 선택한다.
- `TRACK_DEPENDENT`와 `MANUAL_REVIEW`는 범위를 추정하지 않고 `NEEDS_REVIEW`로 반환한다.
- 교사 검증 완료 과목만 후보에 남기고 `P`는 숫자 0이 아닌 원문 라벨로 보존한다.
- trace에는 규칙 버전, 선택 출처·학기·과목과 제외 이유를 기록하되 학생 식별자는 포함하지 않는다.
- 전형별 활성 성적 범위 규칙을 하나만 허용하는 PostgreSQL partial unique index와 게시 규칙 조회 서비스를 추가했다.
- 실제 대학 산식 없이 합성 범위 규칙과 합성 익명 성적만 사용했다.
- 관리자 직접 편집과 CSV import/export가 공유하는 `ManagedScoreRule` canonical 스키마를 추가했다.
- 표준 CSV는 고정 헤더·UTF-8/BOM·TRUE/FALSE·Decimal·빈값/0·중복 업무키를 엄격히 검증한다.
- 자유 수식과 수식형 셀을 거부하고 오류 행과 유효 행을 분리하되 DB 저장이나 부분 게시를 수행하지 않는다.
- 면접·실기 비율은 비예측 메타데이터로만 보존하고 검정고시 변환표는 현재 범위에서 제외했다.
- 복잡한 Z점수 표는 JSON 셀이 아닌 별도 `z_score_tables.csv`와 `z_score_table_code`로 연결한다.
- CSV 업로드·변경 비교·관리자 확인·DRAFT 생성·승인 UI는 Phase 7 경계로 남겼다.
- 전체·최초·최근·우수 N개 학기와 우수 N개 과목을 입력 순서와 무관하게 선택하는 엔진을 추가했다.
- 과목 범위는 명시 코드만 사용하고, 비교 가능한 Decimal 값이 없거나 수동 검토 정책이면 계산하지 않는다.
- 학년별 또는 정확한 학기별 공식 가중치를 선택 후 적용하며 두 방식을 동시에 사용하거나 일부 비율이 비면 오류로 처리한다.
- 면접·실기는 계산에서 제외한 안내 trace로 유지하고 반올림·절사는 선언된 최종 단계에서만 적용한다.
- 참고 계산 엑셀과 제공된 2027 공식 PDF를 읽기 전용으로 대조해 전역 선택과 학년별 우수학기 선택을 분리했다.
- 값의 우수 방향, 전역·학년·학년 내부 학기 가중치, 학기 중간 반올림과 표시 전용 자릿수를 제한 코드로 명시했다.
- 공식 PDF의 Z점수 구간을 표현하도록 하한·상한 포함 여부를 분리하고 계산식·반올림·절단·표 버전 trace를 추가했다.
- 평균등급을 배점으로 바꾸는 공식 구조는 자유 수식 대신 `base + multiplier × 집계값` 제한형 환산만 허용했다.
- 성적 계산 규칙도 전형별 활성 게시 버전을 하나만 허용하며 참고자료를 공식 출처로 위장하지 못하게 검증한다.
- 참고자료 원본·추출본은 Git에 추가하지 않았고 실제 학생 표기가 있는 엑셀 결과 행은 내용 분석에서 제외했다.
- 읽기 전용 참고 XLSX의 실제 수식을 독립 `Decimal` 기준식으로 재현해 합성 등급 500세트를 차등 검증했다.
- 지원 가능한 15개 집계 수식은 7,500건, 이수단위 가중 학기 평균은 2,500건, XLSX 참고 Z점수 경계는 601건 모두 현재 엔진과 일치했다.
- `대학별 등급!J18`은 `E:G` 순위를 `C:G` 값 범위에 적용하는 참조 불일치가 있고, `J20`은 2학년 우수학기와 2학년 2학기를 혼합하므로 자동 규칙화하지 않고 `MANUAL_REVIEW` 대상으로 분리했다.
- XLSX Z점수 음수 경계 일부는 제공된 대학 공식 PDF 경계와 달라 XLSX 계약은 `VERIFIED_REFERENCE`로만 검증했으며 대학별 공식 표를 덮어쓰지 않는다.
- 성취도 등급·분포 변환은 버전 고정 표와 `RATIO`·`PERCENT` 합계를 검증하며 `P`나 빈 분포를 0으로 변환하지 않는다.
- 출결은 검증된 미인정 결석·지각·조퇴·결과와 명시된 환산 단위가 모두 있을 때만 별도 점수를 만들고 교과점수와 분리된 trace로 합산한다.
- 성취도·출결의 참고표를 대학 공식 출처로 위장하지 못하게 차단하고 규칙 CSV에 표 코드·출처·환산 단위를 추가했다.
- Hypothesis로 결정성·입력 순서 독립성·단조성·점수 범위를 검증하고 합성 30/30/40+출결 독립 수기 골든 결과를 고정했다.
- Phase 4 최종 회귀에서 단위 144건, PostgreSQL 통합 29건, Ruff·포맷·mypy·규칙·민감자료 검사를 모두 통과해 게이트를 `PASS`로 판정했다.

## 2026-07-12 · Phase 5 시작

- Phase 5 작업 카드와 5개 파일럿의 문서·학년도·근거 상태 매트릭스를 만들었다.
- 동양미래대·명지전문대·인하공전·연성대는 제공된 2027 공식 모집요강 기반 `UNIVERSITY_OFFICIAL` 검증 후보로 두되 아직 게시하거나 사람 승인하지 않았다.
- 폴리텍 서울정수 자료는 2027 표지와 2026 내부 표시가 혼재해 규칙을 추정하지 않고 `MANUAL_REVIEW`로 분리했다.
- 참고 XLSX와 공식 문서의 구조적 일치·불일치만 기록하고 원본·추출본이나 실제 대학 seed는 추가하지 않았다.
- DB에 저장된 canonical 성적 규칙 payload를 검증 후 동일한 `ScoreRuleDefinition`으로 복원하는 역변환을 추가해 파일럿 실행 연결 기반을 마련했다.
- 파일럿 후보의 연도·근거·자격·성적 범위·계산 payload를 함께 검증하고 `VERIFICATION_PENDING → GOLDEN_PENDING → HUMAN_APPROVAL_PENDING` 게이트를 강제했다.
- 혼합연도 후보는 payload를 실행하지 않고 `MANUAL_REVIEW`로 두며 후보 서비스는 `PUBLISHED` 입력이나 자동 게시를 허용하지 않는다.
- 운영 게시 경로와 분리된 후보 검증 전용 자격·성적범위 진입점을 추가했으며, 이 경로도 자격 허용 전에는 성적 선택을 차단한다.
- 인하공전 공식 PDF에서 일반고 전형의 위탁직업교육 지원 가능, 위탁학기 제외, 최우수 1학기, 200점 환산과 출결표가 연결되어 첫 합성 골든 후보를 작성했다.
- 연성대 표를 이미지로 재검증해 정원내 특별전형 일반고는 직업위탁생 지원 가능, 특성화고는 지원 불가로 열 정렬 오류를 정정했다. 명지전문대는 위탁학기 성적 범위가 불명확해 보류했다.
- 동양미래대는 같은 직업위탁 합성 학생이 일반고 전형에서는 위탁학기 제외, 특성화고 전형에서는 위탁성적 포함 규칙을 적용받도록 후보 골든을 분리했다.
- 우수 2학기 평균과 교과 `950-(50×환산등급)`, 출석 환산을 적용한 후보 결과가 각각 915.00점과 965.00점으로 달라짐을 확인했다.
- 연성대 일반전형의 학기별 이수단위 합 15 미만 제외 조건을 발견해 공통 DSL에 `minimum_semester_credits`를 추가하고 우수학기 선택 전에 적용했다.
- 연성대 정원내 특별전형 일반고는 직업위탁 합성 학생을 조건부 자격으로 두고 위탁학기·15단위 미만 학기를 제외한 뒤 우수 2학기와 출결을 합산하는 965.00점 골든 후보를 확인했다.
- 공식 문서가 있어도 위탁학기 성적 범위가 불명확하면 후보 게이트가 `GRADE_SOURCE_SCOPE_UNRESOLVED`로 수동 검토 처리하도록 보완했다.
- Phase 5는 동양미래대·인하공전·연성대 `VERIFICATION_PENDING`, 명지전문대·폴리텍 서울정수 `MANUAL_REVIEW`로 전부 분류하고 실제 대학 규칙 게시 0건으로 게이트를 통과했다.
- 폴리텍 결합 PDF를 페이지 단위로 다시 분리해 1~32쪽은 서울정수 2027 모집요강, 33~34쪽은 서울정수 2026 입시결과, 35~69쪽은 서울강서 2026 모집요강·입시결과임을 확인하고 기존 혼합연도 판정을 정정했다.
- 서울정수 일반·특별전형의 직업위탁 지원 후보, 교과 320점·출석 80점, 학년 30%·30%·40%와 학기 4자리·학년 2자리 반올림을 합성 골든으로 고정했다.
- 폴리텍 XLSX 수식은 30%·30%·40% 구조는 같지만 공식 중간 반올림이 없어 정밀 합성값에서 XLSX 374.49점·공식식 374.44점으로 달라짐을 회귀 테스트로 고정했다.
- 공식 중간 반올림을 자유 수식 없이 표현하도록 canonical 규칙과 CSV에 `grade_rounding`을 추가했으며, 서울정수 33~34쪽과 서울강서 67~69쪽 결과는 캠퍼스별 2026 과거 입시결과로만 Phase 6에 전달했다.

## 2026-07-12 · Phase 6 시작

- Phase 6 작업 카드와 참고 CSV·노트북의 비식별 용도 분류를 작성했다.
- CSV는 과거 입시결과 자료이며 성적 규칙 입력이 아니고, 100행 파일 두 개는 동일 해시 중복 후보임을 확인했다.
- 네트워크 POST와 파일쓰기가 포함된 기존 노트북은 실행하지 않고 source adapter 계약으로 대체하기로 했다.
- source adapter의 요청 생성·응답 추출·canonical 정규화를 분리하고 timeout·재시도·rate limit·크기·페이지·행 제한을 고정했다.
- raw 요청 fingerprint·응답·collection digest와 staging 행을 분리하고 중복·빈 업무키·혼합연도·행/페이지 급감을 batch 전체 차단한다.
- raw·staging·published 여섯 PostgreSQL 테이블과 migration을 추가하고 오류 없는 staging 전체 행을 관리자가 확인해야만 게시하도록 계약을 고정했다.
- 과거 입시결과에는 해당 모집학년도의 규칙 ID·버전만 선택적으로 연결하며 현재 규칙으로 자동 재해석하지 않는다.
- 결과 분석 서비스는 대학·캠퍼스·학년도·모집시기·전형·학과가 모두 일치하는 활성 `PUBLISHED` 결과만 반환한다.
- Phase 6 최종 회귀에서 단위 174건, PostgreSQL 통합 32건, Ruff·포맷·mypy·migration drift·규칙·민감자료 검사를 모두 통과해 게이트를 `PASS`로 판정했다.


## 확정 기술 결정

- uv로 Python 환경과 잠금 파일을 관리한다.
- PostgreSQL을 기본 DB로 사용하고 `web`·`db`·`postgres_data`를 분리한다.
- Jinja2 SSR을 기본으로 하며 SPA·TypeScript를 사용하지 않는다.
- 핵심 흐름은 JavaScript 없이 동작하고 별도 A4 인쇄 CSS를 사용한다.

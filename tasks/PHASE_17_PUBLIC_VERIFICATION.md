# Phase 17 작업 카드 — 공개 검증 흐름과 2026 입시결과 복구

## 목표

- 직업위탁 재학생을 공개 상담의 기본 대상자로 고정하고 일반고 졸업생만 명시적 예외로 선택한다.
- 1·2학년 원적교 성적과 3학년 위탁 성적을 화면 선택값이 아니라 서버 계약으로 분리한다.
- 3학년 2학기는 선택 입력으로 제공하되 비어 있어도 대학 선택과 적용 가능한 규칙의 계산을 막지 않는다.
- 사용자 제공 공개 자료의 2026학년도 입시결과와 전체 대학·학과 검색 범위를 복구한다.
- 학생·교사·주 관리자·보조 관리자 공개 데모 계정으로 역할별 메뉴, 로그인, 로그아웃을 확인한다.
- 학생·교사 BYOK 키를 계정별로 암호화 저장·마스킹·삭제하며 공개 데모에서는 브라우저 세션별로 격리한다.

## 승인된 변경 경로

- 애플리케이션: `app/`
- DB 마이그레이션: `migrations/`
- 공개 정제 데이터와 출처 레지스트리: `data/seed/`, `data/sources/index.yaml`
- 검증·파생 자동화: `scripts/`, `Makefile`
- 합성 익명 검증: `tests/`
- 문서: `README.md`, `DEVELOPMENT_LOG.md`, `PROJECT_STATUS.md`, 이 작업 카드
- 배포 설정: `.env.example`, 기존 Compose 파일의 데모 환경 전달 항목
- 배포·롤백 검증: `deploy/`

## 금지 사항

- 원본 PDF·XLSX·크롤링 CSV를 Git에 추가
- 2026 입시결과를 2027 모집요강·지원자격·성적 규칙으로 복사
- 비어 있는 3학년 2학기 성적을 0점 또는 결격으로 변환
- 대학명·전형명 문자열만으로 지원자격을 추정
- 공개 데모 관리자에게 운영 데이터 변경 권한 또는 실제 회원 개인정보 노출
- API 키 평문을 DB·로그·HTML·테스트 결과에 기록

## 수용 기준

1. 공개 입력은 1·2학년을 `HOME_SCHOOL_RECORD/FALSE`, 기본 위탁학생의 3학년을 `VOCATIONAL_TRAINING_RECORD/TRUE`로 서버에서 확정하며 출처 드롭다운을 표시하지 않는다.
2. 3학년 2학기 10칸을 추가하고 전부 비어 있어도 검수와 대학 선택이 가능하다.
3. 기본 대상은 직업위탁 재학생이고 예외 선택 시 일반고 졸업생 사실값이 서버에서 일관되게 적용된다.
4. 공개 학생 결과와 출력에서는 공식 자격 판정이 확정 `INELIGIBLE`인 전형을 제외한다.
5. 2026 공개 입시결과가 기본 연도이며 제공 자료의 모든 대학을 이름·학과로 검색하고 선택할 수 있다.
6. 2026 결과는 참고자료로만 표시하고 2027 공식 계산 규칙이 없는 대상은 계산 준비 중으로 명시한다.
7. 네 역할 데모 계정이 로그인 페이지에 표시되고 각 역할 대시보드와 로그아웃이 동작한다.
8. 공개 데모 관리자는 읽기 전용이며 실제 회원·업로드 원문을 열람하거나 변경할 수 없다.
9. 학생·교사 BYOK 키는 서로 다른 actor 범위에 암호화 저장되고 마스킹·교체·삭제가 동작한다. 공개 데모 키는 세션 범위이며 로그아웃 시 삭제된다.
10. 단위·PostgreSQL 통합·Playwright·lint·mypy·규칙·공개 seed 검증을 통과한다.

## 릴리스 게이트

1. 사용자 미추적 파일을 제외한 현재 변경만 검토한다.
2. 공개 2026 파생 seed의 원본 해시·행 수·대학 수·제외 사유를 검증한다.
3. 전체 회귀 후 자연스러운 한국어 커밋으로 `origin/main`에 push한다.
4. 운영 PostgreSQL custom-format 백업과 격리 복원을 검증한다.
5. 기존 DB·volume을 보존하고 `web-production`만 재빌드한 뒤 seed·health·역할별 HTTPS smoke를 확인한다.

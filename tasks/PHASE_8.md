# Phase 8 작업 카드

## 목표

교사가 합성 또는 검수 완료 학생 사실을 단계별로 확인하고, 지원자격이 허용된 전형에 한해서만 성적 범위·계산·같은 연도와 규칙 버전의 입시결과를 조회하도록 상담 SSR 흐름을 구현한다. 학생용은 간결한 결과를, 교사용은 판정·계산·근거 trace와 상담 메모를 포함한 별도 A4 출력으로 제공한다.

## 선행조건

- `PROJECT_STATUS.md`의 Phase 7 `PASS`
- 지원자격 판정 전 성적 조회·계산 차단 계약 유지
- 게시·근거·독립 검증·골든·사람 승인 게이트를 통과한 규칙만 운영 조회
- 과거 입시결과는 같은 모집학년도·업무키·당시 규칙 버전이 연결된 게시 자료만 비교

## 근거 문서·규칙 버전

- `2027_전문대_입시상담앱_실행개발문서_v2.md` FR-08 및 Phase 8
- `CODEX_MASTER_PROMPT_2027_전문대_입시상담앱_v3.md` 9장 및 Phase 8
- 실행 시 사용한 자격·성적 범위·성적 계산 규칙 ID와 버전을 출력 trace에 고정

## 허용 수정 경로

- 상담 orchestration·조회 서비스: `app/services/`
- Flask SSR route·template·CSS·필요 최소 Vanilla JavaScript: `app/`
- 상담 저장 계약이 필요한 경우: `app/models.py`, `migrations/`
- 합성 검증: `tests/`, `e2e/`, `Makefile`
- 기록: `README.md`, `PROJECT_STATUS.md`, `DEVELOPMENT_LOG.md`, `docs/`, `tasks/PHASE_8.md`

## 금지 사항

- 두 기준 원본 문서와 Phase 0~7 카드 수정
- 실제 학생 자료·원본·추출본·상담 메모를 테스트나 Git에 추가
- 지원자격 미확정·불가·검토 필요 전형의 성적 조회 또는 계산
- 다른 모집학년도·다른 산식의 입시결과 직접 비교
- 누락 면접·실기·출결을 0점 또는 만점으로 임의 처리
- 합격 확률이나 안정·적정·소신·위험 자동 판정
- 공식 근거가 없는 규칙 추정 또는 게시
- 핵심 상담·출력 기능을 JavaScript 전용으로 구현

## Given / When / Then 수용 기준

- Given 자격 판정이 완료되지 않은 전형, When 계산을 요청하면, Then 성적 규칙과 성적 행을 조회하지 않고 명시적으로 차단한다.
- Given 자격 허용 전형과 게시 규칙, When 상담 결과를 생성하면, Then 사용한 성적 출처·학기·과목·중간값·반올림·규칙 버전·근거를 재현 가능한 trace로 제공한다.
- Given 같은 연도·업무키·규칙 버전의 게시 입시결과, When 비교를 요청하면, Then 평균·최저·경쟁률·모집인원을 자료 기준연도와 함께 보여준다.
- Given 비교 가능한 게시 입시결과가 없음, When 결과를 표시하면, Then 0이나 추정값 대신 자료 없음·확인 필요를 표시한다.
- Given 하나의 상담 결과, When 학생용과 교사용 출력을 열면, Then 학생용에는 간결한 결과·경고를, 교사용에는 trace·근거·검수 상태·상담 메모를 분리해 표시한다.
- Given JavaScript 비활성 환경, When 단계 입력·결과·출력을 이용하면, Then 핵심 흐름이 서버 렌더링 폼과 링크로 동작한다.
- Given A4 인쇄 미리보기, When Chromium에서 학생용·교사용을 각각 인쇄하면, Then 각 레이아웃이 A4 경계를 넘거나 잘리지 않는다.

## 먼저 작성할 실패 테스트

1. 자격 허용 이전 계산 orchestration 차단
2. 자격 허용 전형만 성적 범위와 계산 실행
3. 규칙 버전이 다른 입시결과 제외
4. 자료 없음과 숫자 0 구분
5. 학생용·교사용 필드 분리와 금지어 미노출
6. JavaScript 없는 단계형 SSR 흐름
7. 학생용·교사용 A4 인쇄 레이아웃과 모바일 overflow

- 구현과 별도로 합성 기대값·규칙 버전·입시결과 업무키를 수기 대조한다.

## 남은 위험

- 실제 공개 규칙은 사람 승인과 게시가 끝난 전형만 화면에 나타나므로 초기 운영 데이터가 비어 있을 수 있다.
- 브라우저별 인쇄 엔진 차이는 Chromium 통과 후 별도 운영 브라우저 검증이 필요하다.
- BYOK 상담 문장 초안은 Phase 9 범위이며 Phase 8 출력에서는 제공하지 않는다.

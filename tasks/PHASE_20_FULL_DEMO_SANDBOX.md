# Phase 20 작업 카드 — 운영 격리형 전체 기능 체험 환경

## 목표

- 학생·교사·보조 관리자·주 관리자 체험계정이 각 역할에 허용된 실제 화면과 저장 흐름을 사용한다.
- 체험 요청은 운영 계정·학생·성적·BYOK·업로드·크롤링 실행 상태를 절대 변경하지 않는다.
- 체험 환경은 별도 애플리케이션 컨테이너, PostgreSQL DB, 업로드 영역, 세션·암호화 키를 사용한다.
- 대학·학과·전형과 2025·2026 입시결과는 검수된 공개 seed를 사용한다.
- 실제 원본이 없는 학생·학급·성적은 체험용임을 표시한 합성 익명 자료만 사용하며 사용자가 추가·수정할 수 있다.
- 이메일·Google·크롤링처럼 외부 부작용이 있는 기능은 체험 전용 outbox 또는 고정 fixture로 같은 사용자 흐름을 검증한다.

## 선행조건과 근거

- Phase 19 이메일 로그인·계정 보안 구현을 보존한다.
- `2027_전문대_입시상담앱_실행개발문서_v2.md`의 개인정보·공식 근거 원칙을 유지한다.
- `CODEX_MASTER_PROMPT_2027_전문대_입시상담앱_v3.md`의 단계 게이트와 비파괴 배포 순서를 따른다.
- 체험 역할은 권한 차이를 유지한다. 보조 관리자는 승인만, 주 관리자는 역할·입시자료·크롤링·규칙 관리까지 허용한다.

## 승인된 변경 경로

- 애플리케이션: `app/`
- DB migration: `migrations/versions/`
- 체험 초기화·운영 자동화: `scripts/`, `docker-compose*.yml`, `deploy/`, `.env.example`, `Makefile`
- 합성 익명 검증: `tests/`, `e2e/`
- 상태·운영 문서: `README.md`, `DEVELOPMENT_LOG.md`, `PROJECT_STATUS.md`, 이 작업 카드

## 금지 사항

- 체험 컨테이너에 운영 DB DSN, 운영 관리자 자격증명, 운영 BYOK 키, SMTP·Google secret 전달
- 공개 비밀번호 계정으로 운영 DB·운영 업로드·실제 크롤러·실제 메일 발송 변경
- 실제 학생 자료나 사용자가 체험 화면에 입력한 값을 Git seed로 저장
- 체험을 이유로 보조 관리자에게 교사 기능·BYOK·역할 변경·자료 등록·크롤링 권한 부여
- 공개 근거가 없는 대학·전형·입시결과를 실제 자료처럼 표시
- 사용자 미추적 파일이나 두 기준 원본 문서 변경

## Given / When / Then 수용 기준

1. 체험 로그인 요청은 별도 체험 서비스와 DB로 이동하며 운영 DB 계정 수·업무 자료는 전후가 동일하다.
2. 학생 체험계정은 성적 학기와 과목을 추가·수정하고 대학을 선택해 계산 결과를 저장·인쇄할 수 있다.
3. 교사 체험계정은 학급·학생·성적을 추가하고 연결 코드를 발급해 학생 계정과 연동하며 상담 초안을 저장할 수 있다.
4. 보조 관리자 체험계정은 승인 대기 계정 승인만 가능하고 학급·성적·상담·BYOK·교내 결과·역할 변경·자료 등록·크롤링은 403이다.
5. 주 관리자 체험계정은 교사 기능과 계정 역할 변경, 입시결과 등록, 공개 seed 기반 자료 조회·fixture 크롤링, PDF·PNG 업로드 검토를 수행할 수 있다.
6. 네 역할 모두 계정 보안 도움말, 비밀번호 변경, 체험 outbox 이메일 인증·재설정, 체험 Google 연결 흐름을 실행할 수 있다.
7. 학생·교사·주 관리자만 BYOK 키 저장·삭제 화면을 사용하며 체험 키는 체험 전용 암호화 키와 DB에만 저장된다.
8. 체험 대학 선택은 검수된 공개 catalog 전체를 제공하고 2025·2026 공개 결과를 함께 조회한다.
9. 체험 초기화 후 공개 seed와 네 기준 계정이 다시 생성되고 이전 체험 입력·업로드는 제거된다.
10. Playwright는 각 역할의 화면 노출뿐 아니라 POST 이후 DB 저장 결과와 역할별 금지 동작을 검증한다.

## 먼저 작성할 실패 테스트

- 운영·체험 DSN/secret/volume 분리 구성 테스트
- 체험 진입 시 경로 prefix와 별도 session cookie 테스트
- 학생·교사·주 관리자의 저장·수정·연동과 보조 관리자의 승인 전용 권한 경계 테스트
- 2026 원자료 42개 대학·1,048개 학과와 기존 2025 자료를 합친 최종 43개 대학·1,079개 학과·5,092개 전형 및 2025·2026 결과 조회 테스트
- 체험 이메일 outbox·Google 연결·크롤링 fixture 테스트
- 실제 브라우저의 학생/교사 교차 세션 연결 및 관리자 흐름 테스트

## 운영 게이트

- 운영 DB custom-format 백업·checksum·archive·격리 복원
- 운영 DB 불변 기준값 기록 후 체험 DB 초기화·행동 테스트·재확인
- 체험 서비스는 loopback 포트와 `/demo/` reverse proxy로만 노출
- 구현 커밋·push 후 운영 DB volume을 유지한 운영·체험 컨테이너 비파괴 재빌드

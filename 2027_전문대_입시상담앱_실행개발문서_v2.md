# 2027학년도 전문대 입시 상담 앱 실행 개발 문서 v2

- 기준일: 2026-07-10
- 상태: 개발 착수 기준선 확정
- 대상: 고등학교 3학년 직업위탁 학생을 상담하는 교사·관리자
- 최우선 가치: 지원자격 정확성, 계산 재현성, 근거 추적성, 개인정보 최소화

## 1. 프로젝트 판정

판정: 채택

이 앱은 합격 예측기가 아니라 다음 순서를 강제하는 교사용 상담 보조 시스템이다.

```text
공식 문서와 규칙 버전 확인
→ 학생의 원적교·위탁 사실로 전형별 지원자격 판정
→ 지원 가능 전형에 한해 해당 전형의 성적 범위 선택
→ 대학별 환산점수 계산
→ 같은 연도·같은 산식의 입시결과와 비교
→ 학생용·교사용 근거 포함 출력
```

학생을 일반고 또는 특성화고 전형 중 하나로 미리 분류하지 않는다. 동일한 일반고 직업위탁생도 대학에 따라 일반고 전형, 특성화고 전형, 두 전형 모두 또는 확인 필요가 될 수 있다.

## 2. 확정 범위

### 2.1 포함

- 2027학년도 서울·경기·인천 전문대학
- 수도권 한국폴리텍대학 2년제 학위과정
- 대학·캠퍼스·학과·모집시기·전형별 지원자격
- 대학별 학생부 반영 규칙과 계산 과정
- 원적교 1·2학년 성적과 직업위탁기관 3학년 성적의 분리 입력
- PDF, 스캔 PDF, 이미지, 클립보드 이미지, XLSX, XLS, CSV, 표 텍스트 입력
- 문서·규칙·입시결과의 연도 및 버전 관리
- 학생용 A4와 교사용 A4 출력
- 기존 크롤러와 과거 입시결과의 검수형 재사용
- 선택적 BYOK AI 상담 문장 초안

### 2.2 제외

- 합격 확률, 안정·적정·소신·위험 자동 판정
- 생성형 AI의 독자적인 자격 판정이나 점수 계산
- 원서 접수 대행
- 학생 공개 회원가입
- 4년제 대학과 비수도권 전문대 확대
- 폴리텍 비학위과정
- 공식 근거가 없는 규칙의 자동 추정

## 3. 현재 자료 상태

### 3.1 분석 완료 자료

- 공통 기준: 전문대학 입학전형 기본사항
- 대학·학과 참고: 전공별 개설대학 입학상담 자료집
- 최종 모집요강: 동양미래, 명지전문, 한양여대, 경민, 인하공전, 숭의여대, 배화여대, 연성대
- 시행계획: 서일, 경복, 동서울, 재능
- 폴리텍: 서울정수 2027 부분
- 참고표: Z점수 조견표
- 기존 업무 자료: 크롤링 노트북, 수집 XLSX, 필터 XLSX, 상담 계산 엑셀
- 학생 입력 구조: 이미지형 생활기록부와 위탁기관 학급 성적 일람표의 구조

### 3.2 특별 주의

- 폴리텍 통합 PDF는 서울정수 2027과 서울강서 2026이 섞여 있다. 페이지 단위 학년도 검증 없이는 게시할 수 없다.
- 서일·경복·동서울·재능은 최종 모집요강이 아니라 시행계획 기준이므로 사용자에게 임시자료임을 표시한다.
- 동서울 문서의 2026 입시결과 산식과 2027 계획 산식이 다르므로 과거 결과에는 해당 연도 규칙 버전을 연결한다.
- 상담 자료집과 Z점수표는 참고·테스트 자료이며 대학별 최종 규칙의 대체 근거가 아니다.
- 아직 개별 검증하지 않은 CSV와 추가 대학 문서는 `PENDING_ANALYSIS`로 두고 공개 계산에 사용하지 않는다.

### 3.3 문서 신뢰 순위

```text
AMENDED_FINAL_GUIDE
> FINAL_GUIDE
> AMENDED_IMPLEMENTATION_PLAN
> IMPLEMENTATION_PLAN
> COMMON_STANDARD
> REFERENCE_ONLY
> AI_EXTRACTED_DRAFT
```

문서 파일명만 신뢰하지 않고 본문 학년도·캠퍼스·공고일·정정 여부를 검사한다.

## 4. 핵심 도메인 원칙

### 4.1 자격 판정이 계산보다 먼저다

지원자격 상태:

- `ELIGIBLE`
- `CONDITIONALLY_ELIGIBLE`
- `INELIGIBLE`
- `NEEDS_REVIEW`
- 내부 상태 `INSUFFICIENT_DATA`

`INELIGIBLE`이면 계산하지 않는다. `NEEDS_REVIEW`와 `INSUFFICIENT_DATA`는 확정 추천처럼 표시하지 않는다.

### 4.2 학생 정보에는 사실만 저장한다

- 원적교 유형
- 최종 졸업고교 유형
- 졸업·졸업예정 상태
- 직업위탁 참여·이수·예정 상태
- 직업위탁 기간·학기·시간
- 전학 여부
- 검정고시 여부
- 필요한 경우에만 전형별 추가 조건

학교 유형과 위탁 여부를 조합해 각 전형을 독립 판정한다.

### 4.3 학생 성적은 출처별로 분리한다

- 1·2학년: `HOME_SCHOOL_RECORD`
- 3학년 위탁: `VOCATIONAL_TRAINING_RECORD`
- 검정고시: `GED_RECORD`
- 교사 보정: `MANUAL_INPUT`

전형별 규칙이 어떤 출처와 학기를 포함할지 결정한다. 저장 시 두 출처를 하나의 학생부로 덮어 합치지 않는다.

성적 범위 정책:

- `HOME_ONLY`
- `VOCATIONAL_INCLUDED`
- `VOCATIONAL_ONLY`
- `EXCLUDE_VOCATIONAL_SEMESTER`
- `TRACK_DEPENDENT`
- `MANUAL_REVIEW`

### 4.4 계산은 제한형 규칙 DSL로 표현한다

대학명을 검사하는 전용 함수나 `eval()` 가능한 수식 문자열을 사용하지 않는다. 표현이 불가능한 사례는 `custom_strategy`와 ADR·근거·골든 테스트를 요구한다.

### 4.5 모든 판정과 계산은 설명 가능해야 한다

- 규칙 ID·버전·상태
- 선택한 성적 출처·학기·과목
- 변환표와 경계값
- 이수단위 가중치
- 반올림·절사 단계
- 출결·면접·실기·가감점
- 최종값과 만점
- 근거 문서·쪽·표 위치
- 검수자·승인 상태

## 5. 기능 요구사항

### FR-01 문서 레지스트리

- PDF·이미지·XLSX·CSV를 등록한다.
- 해시, 학년도, 대학, 캠퍼스, 문서 종류, 공고일, 수정본, 대체 관계를 저장한다.
- 페이지별 학년도 불일치와 혼합연도 문서를 차단한다.
- `DRAFT → EXTRACTED → VERIFIED → HUMAN_APPROVED → PUBLISHED → SUPERSEDED` 생명주기를 적용한다.

### FR-02 학생 자료 입력 게이트웨이

입력 형식:

- 텍스트 PDF
- 이미지형·손상 PDF
- PNG·JPG·JPEG
- 브라우저 클립보드 이미지 붙여넣기
- XLSX·XLS·CSV
- 표 텍스트 붙여넣기
- 직접 입력

처리 흐름:

```text
입력 수신
→ 파일 형식·중복 해시 판별
→ 문서 유형 분류
→ 표 블록 탐지
→ 표준 필드 변환
→ 신뢰도와 누락 필드 표시
→ 교사 미리보기·수정
→ 확정 행만 저장
→ 원본 자동 파기
```

생활기록부는 고정 페이지가 아니라 OCR로 `교과학습발달상황` 머리글을 찾는다. `세부능력 및 특기사항`은 점수 계산 입력에서 제외한다.

### FR-03 위탁기관 학급 성적표 처리

위탁기관 파일은 학급 전체 학생을 포함할 수 있다.

- 대상 학생 행과 과목 통계만 메모리에서 추출한다.
- 다른 학생 행은 DB·로그·캐시에 저장하지 않는다.
- 정기시험+수행평가형, 능력단위 100%형, 한 시트 내 복수 과목 블록을 지원한다.
- 평균·표준편차·수강자수가 없으면 임의 생성하지 않는다.
- 학점·석차등급이 없으면 `NOT_CALCULABLE` 또는 전형별 대체 규칙을 적용한다.

### FR-04 지원자격 판정

- 모집학년도·대학·캠퍼스·학과·모집시기·전형별로 판정한다.
- 일반고, 특성화고, 마이스터고, 종합고 직업계열, 검정고시, 평생교육시설, 외국고를 구분한다.
- 직업위탁생의 원적교 기준·교육과정 기준·학과 예외를 표현한다.
- 일반고·특성화고 전형 동시 지원 가능성을 허용한다.
- 학교폭력 등 민감 결격조건은 별도 규칙으로 두고 영구 저장을 최소화한다.

### FR-05 성적 계산

- 전체·최근·우수 N개 학기
- 학년별 30/30/40, 40/60 등 가중치
- 전체·특정 교과·우수 N개 과목
- 이수단위 가중·동일 가중
- 석차등급, Z점수, 성취도 분포비율, 검정고시 변환
- 진로선택 포함·제외
- 위탁학기 포함·제외
- 출결·면접·실기·가감점
- 단계별 반올림·절사
- `Decimal` 기반 결정적 계산

### FR-06 복수지원과 결격 규칙

대학별 복수지원 횟수·금지 조합·캠퍼스 제한을 `multiple_application_rules`로 관리한다. 지원자격 제한과 점수 감점은 각각 `disqualification_rules`, `score_adjustment_rules`로 분리한다.

### FR-07 입시결과·크롤링

기존 노트북을 재사용 가능한 수집 어댑터로 전환한다.

```text
raw: 원본 응답·원본 행, 수정 금지
→ staging: 열 정규화·대학/학과/전형 매핑·중복 후보
→ published: 검수·승인·연도·산식 버전 연결 완료
```

- 수동 페이지 수를 제거하고 마지막 페이지를 검증한다.
- 수집 행 수·페이지 수·대학 수를 실행 기록에 남긴다.
- 과거 합격등급에는 당시 `score_rule_version_id`를 연결한다.
- 산식이 다른 연도의 등급을 현재 산식과 직접 비교하지 않는다.

### FR-08 상담 화면과 출력

학생용:

- 대학·학과·전형
- 자격 상태
- 환산점수와 적용 학기
- 과거 평균·최저·경쟁률·모집인원
- 자료 기준연도와 확인 필요 경고

교사용:

- 학생용 정보 전체
- 판정·계산 trace
- 문서·쪽 근거
- 데이터·규칙 검수 상태
- 상담 메모와 후보 학과
- 선택적 AI 문장 초안

학생용과 교사용 A4 인쇄 레이아웃을 별도로 제공한다.

### FR-09 BYOK AI

- OpenAI, Gemini, Anthropic 공급자 어댑터를 허용한다.
- API 키는 서버 측 암호화·마스킹·삭제를 지원한다.
- 학생 이름, 생기부 원문·이미지, 학번, 건강·가정정보를 전송하지 않는다.
- AI는 검증된 계산 결과의 문장화만 수행한다.
- AI가 자격·점수·합격 가능성을 생성할 수 없다.

## 7. 개인정보·보안 요구사항

- 실제 생활기록부·위탁성적 원본을 테스트 픽스처로 사용하지 않는다.
- 합성 익명 데이터만 골든·E2E 테스트에 사용한다.
- 입력 원본의 기본 보존기간은 추출·검수 세션 종료까지다.
- 원본 자동삭제 실패는 오류로 기록하고 관리자에게 표시한다.
- OCR·파싱은 로컬 우선이며 외부 AI 전송은 기본 금지다.
- 로그에 파일명 속 실명, 학생 행, 성적 원문, 비밀키를 남기지 않는다.
- CSRF, 세션 쿠키, RBAC, 감사 로그, 보유기간 삭제 작업을 구현한다.
- 상담 저장 기본값은 비저장 또는 익명 상담 코드다.

| 학생자료 입력 에이전트 | PDF/OCR/클립보드/XLSX/CSV 파서와 개인정보 파기 | 점수 규칙 해석 |
| 지원자격 분석 에이전트 | 자격 후보·반례·근거 연결 | 앱 코드 수정 |
| 성적규칙 분석 에이전트 | 성적 범위·산식·경계·골든 후보 | 자격 미검증 전형 발행 |
| 입시결과 에이전트 | 크롤러 어댑터·raw/staging/published·정규화 | 원본 행 덮어쓰기 |
| 독립 검증 에이전트 | 원문 대조, 반례, 경계값, 개인정보·삭제 확인 | 구현자의 설명만 신뢰 |
| 도메인·백엔드 에이전트 | 순수 Python 엔진, 서비스, DB, TDD | 대학명 하드코딩 |
| UI·출력 에이전트 | Jinja/JS/Tailwind, A4, Playwright | 프론트에서 공식 재구현 |
| 보안·릴리스 에이전트 | RBAC, 키, 로그, 배포·복구 | 기능 범위 확장 |

규칙 추출자와 독립 검증자, 구현자와 최종 QA를 분리한다. 같은 파일을 둘 이상의 에이전트가 동시에 수정하지 않는다.

## 9. TDD 전략

모든 구현은 `Red → Green → Refactor`를 기본으로 한다.

표준 루프:

```text
근거·요구사항 확인
→ 한 개 수용 기준 선택
→ 실패 테스트 작성·실패 확인
→ 최소 구현
→ 리팩터링
→ 대상 테스트
→ 전체 회귀
→ 독립 검증
→ 상태·근거 기록
```

우선순위:

1. 자격·계산 순수 함수 단위 테스트
2. 규칙 JSON 계약 테스트
3. 공식 예시·독립 수기 계산 골든 테스트
4. Hypothesis 속성 테스트
5. 파서 구조·개인정보 파기 테스트
6. DB·서비스 통합 테스트
7. Playwright E2E·A4 시각 테스트

## 10. 필수 테스트 계획

### 10.1 지원자격

- 동일 일반고 위탁생을 동양미래·경복·인하공전·재능·연성 규칙에 넣어 서로 다른 결과 확인
- 일반고·특성화고 두 전형 모두 가능
- 마이스터고·종합고 직업계열 포함·제외
- 검정고시
- 학과별 예외
- `INSUFFICIENT_DATA`와 `NEEDS_REVIEW` 구분
- 지원 불가 시 계산 차단

### 10.2 입력 파서

- 동일 합성 성적표의 PDF·이미지·클립보드·XLSX·CSV 결과 동등성
- 이미지형·손상·여러 페이지 PDF
- `교과학습발달상황` 페이지 위치 변화
- 선택적 `성취도별 분포비율` 열
- `P`와 빈 값
- 한 시트의 복수 과목 블록
- 정기시험+수행평가형과 능력단위형
- 대상 학생 외 학급 행 비저장
- 파일 해시 중복 탐지
- 원본 자동삭제

### 10.3 계산

- 우수 1·2학기, 첫 4학기·5학기 범위, 30/30/40
- 위탁학기 포함·제외·전형별 상이
- Z 경계 1.76, 1.23, 0.74, 0.26, -0.26, -0.74, -1.23, -1.76
- Z 범위 ±3, 표준편차 0·누락
- 진로선택 분포비율 포함·제외
- 단계별 반올림·절사
- 출결·면접·실기·감점
- 결정성·단조성·순서 독립성·점수 범위

### 10.4 문서·데이터 품질

- 혼합연도 자동 게시 차단
- 시행계획의 임시 배지
- 정정본의 이전 버전 대체
- 2026 결과와 2027 규칙 혼용 차단
- 수집 페이지·행 수 급감 탐지
- 공개 규칙의 출처·쪽·골든·승인 100%

### 10.5 E2E

- 파일 업로드/붙여넣기 → 미리보기 → 수정 → 확정
- 자격 판정 → 지원 가능한 전형만 계산
- 전형에 따라 위탁성적 범위가 바뀜
- 계산 trace와 원문 근거 표시
- 학생용·교사용 A4 인쇄
- 원본 삭제 상태 표시
- BYOK 없음에도 핵심 기능 정상

### 10.6 품질 게이트

- 도메인 핵심 분기 커버리지 95% 이상
- 전체 분기 커버리지 85% 이상
- 공개 규칙 골든 테스트·출처 보유율 100%
- 핵심 E2E 100% 통과
- Critical·High 결함 0
- 실제 개인자료가 테스트·로그·Git에 0건

## 11. 구현 순서

## 13. Definition of Done

작업 단위:

- 실패 테스트가 먼저 존재하고 실패를 확인했다.
- 공식 근거 또는 명시적 합성 테스트 근거가 있다.
- 대상·회귀 테스트가 통과한다.
- 대학명 하드코딩이 없다.
- 누락을 0점·불가로 바꾸지 않는다.
- 판정·계산 trace가 재현된다.
- 개인자료가 로그·픽스처·Git에 없다.
- 독립 검증이 `PASS`다.
- 상태 문서와 작업 카드가 갱신됐다.

릴리스:

- 대상 전형이 공개·임시·차단·범위제외 중 하나로 완전히 분류됐다.
- 공개 규칙의 출처·쪽·골든·승인 100%다.
- 핵심 상담·업로드·인쇄 E2E가 통과한다.
- Critical·High 결함이 없다.
- 삭제·보유기간·정정·백업·복구 절차가 검증됐다.
- 교사가 최종 공개 규칙을 승인했다.

## 14. 최초 실행 작업

1. 저장소와 사용자 변경을 보존한 채 실제 파일 인벤토리를 다시 만든다.
2. 개인 생활기록부·위탁성적 원본과 파생 이미지가 남지 않았는지 검사한다.
3. 문서 레지스트리 스키마와 혼합연도 차단 테스트를 작성한다.
4. 학생 성적 출처 모델과 XLSX·CSV 입력 계약 실패 테스트를 작성한다.
5. 지원자격 규칙 DSL과 일반고 위탁생 교차 대학 테스트를 작성한다.
6. 성적 범위 정책과 계산 엔진 골든 테스트를 작성한다.
7. 파일럿 5개 대학을 출처→규칙→테스트→사람 승인 대기까지 연결한다.
8. 이후에 상담 UI와 입시결과 비교를 구현한다.

화면부터 만들지 않는다. 첫 수직 슬라이스의 완료 기준은 `입력 → 자격 → 성적 범위 → 계산 → 근거 → 삭제 확인`의 전 과정이 합성 학생 데이터로 통과하는 것이다.

# Phase 16 역할별 업무공간과 학생·교사 연결

## 역할 경계

| 역할 | 허용 업무 |
|---|---|
| `STUDENT` | 본인 성적·상담 관리, 교사 학급 연결, 공유자료 읽기, 본인 BYOK 키·상담 문장 |
| `TEACHER` | 본인 학과·학급·비식별 명단과 성적, 연결 학생 자료 읽기, 상담자료 저장·BYOK 초안, 교내 지원 결과 |
| `ASSISTANT_ADMIN` | 승인 대기 `MEMBER`·`STUDENT`·`TEACHER` 승인 |
| `ADMIN` | 승인·역할·상태, 기준정보·규칙, 입시결과 import·제한형 수집, 근거 문서 등록·검증 |

대시보드 메뉴와 서버 route 권한은 같은 역할표를 사용한다. 보조 관리자는 상담·BYOK·입시결과·근거 문서·수집 route에 접근할 수 없다.

## 연결과 공유

1. 교사는 학년도·학과·학급과 ASCII 비식별 학생 코드를 만든다.
2. 서버는 144비트 이상 난수의 연결 코드를 한 번만 표시하고 SHA-256 digest·끝 4자리·24시간 만료시각만 저장한다.
3. 학생은 연결 화면에서 공유 범위를 확인하고 명시적으로 동의한 뒤 코드를 입력한다.
4. 연결 중 교사는 학생 계정 소유 성적·상담을 읽고, 학생은 교사 관리 성적·저장 상담을 읽는다.
5. 상대 소유 자료는 수정·삭제할 수 없으며 DB 레코드를 복제하거나 소유권을 이전하지 않는다.
6. 학생에게는 공유 상담의 학생용 결과만 제공하고 교사용 A4·상담 메모는 노출하지 않는다. 상대 소유 상담은 입력 화면으로 복제할 수 없다.
7. 학생 연결 해제, 교사 코드 재발급, 학생·교사 정지 또는 역할 변경 시 기존 공유·미사용 코드가 폐기된다.

연결 동의 범위는 해당 학생 계정에 저장된 전체 성적·상담자료이며 화면에 이를 명시한다. 이름·이메일로 자동 연결하지 않는다. 연결·해제·코드 회전·계정 변경에 따른 폐기는 감사 이벤트로 남기되 코드 원문과 학생 개인정보는 기록하지 않는다.

## 상담자료와 BYOK

교사는 학급 학생의 교사 입력 성적 또는 연결 학생의 계정 소유 성적을 선택해 지원자격 우선 계산을 수행한다. 결과 화면의 명시적 POST로 학생 참조에 상담자료를 저장하며 연결 학생에게 읽기 전용으로 공유된다. 연결 해제 뒤에는 교사 소유 저장본은 유지되지만 학생 화면에서는 즉시 사라진다.

학생과 교사는 계정별 암호화 BYOK 키를 사용한다. 외부 AI에는 기존 비식별 allowlist payload만 전달하고 AI가 지원자격·점수·추천·순위·합격 가능성을 생성하게 하지 않는다. 공급자 키가 없어도 결정론적 상담 계산·저장·A4 출력은 동작한다.

## 관리자 입력

주 관리자의 입시결과 화면은 CSV·XLSX preview/review/publish와 상한이 고정된 전문대학포털 수집을 제공한다. 근거 문서 화면은 PDF·PNG·JPG·CSV·XLSX의 signature·크기·내용·hash를 검증하며 손상 PDF/XLSX와 ZIP 내부 OOXML이 손상된 XLSX는 HTTP 400으로 처리한다. 업로드나 수집 결과는 자동으로 공식 규칙 또는 입시결과로 게시하지 않는다.

## 검증

- PostgreSQL 통합: 역할별 메뉴·403, 연결 코드 digest/만료/재사용 차단, 양방향 읽기·상대 수정 차단, 상담 저장·해제, 계정 정지 폐기, 손상 업로드
- Playwright: 주/보조 관리자와 학생/교사 메뉴 격리, 교사 학급·학생·성적 생성, 학생 동의·연결·읽기 전용 표시, JavaScript 비활성 SSR
- 공통 게이트: Ruff·format·mypy, 단위·전체 PostgreSQL, migration drift, 규칙 검증, 민감자료 검사, 백업·격리 복원

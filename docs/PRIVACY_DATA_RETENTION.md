# 개인정보와 자료 보유

- 실제 학생 파일은 Git, Docker 이미지, 테스트 픽스처, 로그에 저장하지 않는다.
- 업로드 원본은 추출·검수 세션 종료까지만 임시 보관하고 삭제 결과를 검증한다.
- 다른 학생의 행은 저장하거나 로그에 기록하지 않는다.
- OCR과 파싱은 로컬 우선이며 외부 AI 전송은 기본 금지한다.
- API 키는 `.env`에도 평문 장기 보관하지 않는 방향으로 Phase 9에서 암호화 저장을 구현한다.

## Phase 1 삭제 계약

- 검수 세션과 파일에는 원본 파일명 대신 무작위 식별자를 사용한다.
- 원본과 파생물은 `original`, `derived`로 분리하되 같은 세션 폐기 시 함께 삭제한다.
- 삭제 호출 뒤 세션 경로가 남으면 `DeletionVerificationError`로 실패를 표면화한다.
- 테스트에는 합성 바이트만 사용하며 학생 이름·성적 원문을 기록하지 않는다.

# 개인정보와 자료 보유

- 실제 학생 파일은 Git, Docker 이미지, 테스트 픽스처, 로그에 저장하지 않는다.
- 업로드 원본은 추출·검수 세션 종료까지만 임시 보관하고 삭제 결과를 검증한다.
- 다른 학생의 행은 저장하거나 로그에 기록하지 않는다.
- OCR과 파싱은 로컬 우선이며 외부 AI 전송은 기본 금지한다.
- API 키는 `.env`에도 평문 장기 보관하지 않는 방향으로 Phase 9에서 암호화 저장을 구현한다.

## Phase 1 삭제 계약

- 검수 세션과 파일에는 원본 파일명 대신 무작위 식별자를 사용한다.
- 원본과 파생물은 `original`, `derived`로 분리하되 같은 세션 폐기 시 함께 삭제한다.
- 삭제 호출 뒤 세션 경로가 남으면 `DeletionVerificationError`로 실패를 표면화한다.
- 테스트에는 합성 바이트만 사용하며 학생 이름·성적 원문을 기록하지 않는다.

## Phase 2 확정 저장 계약

- 교사가 명시적으로 선택한 표준 행만 PostgreSQL에 저장한다.
- 학급표의 대상 학생 식별자는 메모리 비교에만 사용하며 다른 학생 행은 반환·로그·DB에 남기지 않는다.
- DB 입력을 먼저 검증·flush하되 임시 원본과 파생물 삭제가 확인된 뒤에만 commit한다.
- 삭제 실패 시 DB transaction을 rollback하고 오류를 조용히 무시하지 않는다.
- 원본 삭제 성공 뒤 DB commit 실패 시에는 개인정보 최소 보유를 우선하며 삭제 파일을 복구하지 않는다.
- 텍스트 PDF 추출은 로컬 메모리에서만 수행하고 원문 페이지 텍스트를 로그·DB에 저장하지 않는다.
- `세부능력 및 특기사항` 머리글 이후 내용은 정규화·검수·저장 대상에서 제외한다.
- 이미지·클립보드 OCR은 컨테이너 내부 Tesseract에서만 수행하고 외부 서비스에 이미지나 OCR 원문을 보내지 않는다.
- OCR 전체 텍스트와 Tesseract stderr는 로그·DB에 저장하지 않고 표준 필드 후보만 메모리에 유지한다.
- 이미지형 PDF 렌더링 이미지는 페이지별 메모리 객체로만 사용하고 OCR 직후 닫으며 파일이나 캐시에 기록하지 않는다.
- 세부능력 및 특기사항이 인식된 뒤의 PDF 페이지는 추가 렌더링·OCR하지 않는다.
- 검수 상태는 무작위 세션 경로의 권한 `0600` 파일에만 임시 보관하고 2 MiB를 넘기지 않는다.
- 검수 화면에는 원본 파일명과 학생 식별자를 렌더링하지 않는다.
- 검수·완료 응답은 `Cache-Control: no-store`, `Referrer-Policy: no-referrer`, CSP와 frame 차단을 적용한다.
- 행 확정과 세션 폐기는 CSRF 검증 POST만 허용하며 검증 실패 시 세션을 삭제하거나 DB에 저장하지 않는다.

## Phase 3 민감 결격 입력

- 학교폭력 등 민감 결격 사실은 영구 학생 사실 모델이나 규칙 payload에 저장하지 않는다.
- 결격 판정 함수에는 제한된 이름의 bool 사실만 요청 메모리로 전달한다.
- 설명 trace에는 사실 이름·조건 결과만 남기고 실제 bool 값은 복사하지 않는다.
- 결격 사실 누락은 허용 또는 결격으로 추정하지 않고 `INSUFFICIENT_DATA`로 처리한다.
- 결격 결과를 점수 감점으로 변환하거나 `score_adjustment_rules`에 전달하지 않는다.

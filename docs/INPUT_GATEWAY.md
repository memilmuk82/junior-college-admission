# 학생 자료 입력 게이트웨이

## 단계와 안전 경계

Phase 2는 CSV·XLSX·표 붙여넣기 계약을 먼저 안정화한 뒤 PDF·이미지 OCR과 교사 검수 화면으로 확장한다. 현재 구현은 CSV·표 붙여넣기와 XLSX의 메모리 내 미리보기이며 PostgreSQL 저장이나 성적 계산을 수행하지 않는다.

정규화 결과에는 다음만 포함한다.

- 원본 바이트를 식별하기 위한 SHA-256
- 입력 형식
- 알려진 표준 필드 후보
- 누락·잘못된 형식의 행 번호와 필드 코드
- 값이 저장되지 않은 알 수 없는 열의 머리글

알 수 없는 열의 셀 값과 입력 원문 전체는 반환하거나 로그에 남기지 않는다. 빈칸과 선택 열 부재는 `None`으로, 이수 표기 `P`는 `P`로 유지하며 숫자 `0`으로 바꾸지 않는다.

## 현재 표준 필드

- 학년도, 학년, 학기
- 교과·교과군, 과목·과목명
- 학점·이수단위
- 원점수, 평균, 표준편차
- 성취도, 수강자수, 석차등급

이 계약은 입력 미리보기용이다. 교사가 확인한 행만 저장하는 계약은 후속 Phase 2 작업 단위에서 별도 실패 테스트 후 연결한다.

## XLSX 처리 제한

- 최대 원본 크기: 20 MiB
- 최대 시트 수: 20개
- 시트별 머리글 탐색: 처음 50행
- 시트별 데이터 처리: 최대 5,000행
- 읽기 전용·수식 결과값 모드이며 외부 workbook 링크는 읽지 않음

시트마다 과목 필드를 포함한 표준 머리글을 탐색한다. 머리글을 찾지 못한 시트는 임의 해석하지 않고 `HEADER_NOT_FOUND` 검수 이슈로 남긴다.

## 학급 성적표 대상 학생 격리

CSV·표 붙여넣기·XLSX 학급표의 `학생식별자` 또는 `student_reference` 열은 대상 행을 메모리에서 비교하는 용도로만 사용한다. 반환되는 정규화 행에는 대상 식별자를 포함하지 않으며 다른 학생 행은 표준 필드 변환이나 검수 이슈 생성 대상으로도 사용하지 않는다.

대상 행에 함께 있는 과목 평균·표준편차·수강자수는 누락된 값을 만들지 않고 그대로 보존한다. 대상 학생이 없으면 다른 학생 행을 반환하지 않고 `TARGET_NOT_FOUND`를 표시한다.

## 확인 행 저장과 원본 파기

교사가 선택한 행에는 학년도·학년·학기·과목이 모두 있어야 한다. 선택되지 않은 행은 저장하지 않으며 선택 인덱스 중복이나 범위 오류도 저장 전에 거부한다.

저장 transaction은 다음 순서를 따른다.

```text
선택 행 검증
→ ImportBatch·학기·과목 행 flush
→ 임시 원본·파생물 삭제 검증
→ batch CONFIRMED와 삭제 시각 기록
→ DB commit
```

삭제 검증이 실패하면 DB 변경을 rollback하고 `DeletionVerificationError`를 호출자에게 전달한다. 삭제가 성공한 뒤 DB commit 자체가 실패하는 경우에는 개인정보 최소 보유를 우선해 삭제된 원본을 복원하지 않는다.

원점수의 `P`는 `raw_score`에 0을 넣지 않고 `raw_score_label='P'`로 보존한다. DB CHECK 제약은 숫자 점수와 라벨의 동시 저장 및 `P` 이외의 임의 라벨을 차단한다.

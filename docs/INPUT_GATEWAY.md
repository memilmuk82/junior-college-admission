# 학생 자료 입력 게이트웨이

## 단계와 안전 경계

Phase 2는 CSV·XLSX·표 붙여넣기 계약을 먼저 안정화한 뒤 PDF·이미지 OCR과 교사 검수 화면으로 확장한다. 현재 첫 작업 단위는 CSV와 표 붙여넣기의 메모리 내 미리보기이며 PostgreSQL 저장이나 성적 계산을 수행하지 않는다.

정규화 결과에는 다음만 포함한다.

- 원본 바이트를 식별하기 위한 SHA-256
- 입력 형식
- 알려진 표준 필드 후보
- 누락·잘못된 형식의 행 번호와 필드 코드
- 값이 저장되지 않은 알 수 없는 열의 머리글

알 수 없는 열의 셀 값과 입력 원문 전체는 반환하거나 로그에 남기지 않는다. 빈칸과 선택 열 부재는 `None`으로, 이수 표기 `P`는 `P`로 유지하며 숫자 `0`으로 바꾸지 않는다.

## 현재 표준 필드

- 학년도, 학년, 학기
- 교과·교과군, 과목·과목명
- 학점·이수단위
- 원점수, 평균, 표준편차
- 성취도, 수강자수, 석차등급

이 계약은 입력 미리보기용이다. 교사가 확인한 행만 저장하는 계약과 다른 학생 행 제거는 후속 Phase 2 작업 단위에서 별도 실패 테스트 후 연결한다.

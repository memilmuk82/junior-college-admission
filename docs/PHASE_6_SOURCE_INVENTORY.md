# Phase 6 참고자료 분류

참고자료는 `tmp/codex-reference`에서 읽기 전용으로 확인했으며 Git 입력 파일로 사용하지 않는다.

## CSV

| 파일군 | 내부 연도 | 행 수 | 분류 |
|---|---|---:|---|
| `procollege_all_2026` | 2016~2026 혼합 | 45,385 | 과거 입시결과 통합 참고자료 |
| `result2026` | 2026 | 4,970 | 2026학년도 입시결과 참고자료 |
| `result_2026` 복사본 2개 | 2026 | 각 100 | 동일 SHA-256 중복 후보 |

공통 헤더는 모집학년도·지역·대학명·모집시기·전공명·입학정원·주야·전형구분·출신교·점수산출 구분·경쟁률·평균·최저 결과다. 학교별 성적 규칙 CSV가 아니며 Phase 4 규칙 입력으로 사용하지 않는다.

## 노트북

- 코드 셀 11개, 출력 항목 8개다.
- `requests`, `BeautifulSoup`, `pandas`, `os`, `time`을 사용한다.
- 네트워크 POST, 디렉터리 생성, CSV 파일쓰기가 포함되어 있어 실행하지 않았다.
- 비밀키·환경변수 접근 패턴은 발견되지 않았지만 실제 사이트 호출은 별도 adapter와 승인된 운영 절차로 대체해야 한다.

## Phase 6 처리 원칙

- 기존 CSV는 계약 탐색과 비식별 품질 통계에만 사용한다.
- 동일 해시 복사본은 하나의 수집 원본으로 묶고 중복 게시하지 않는다.
- 2016~2026 혼합 파일은 행별 모집학년도를 유지하며 2027 데이터로 승격하지 않는다.
- crawler 테스트는 합성 fixture와 가짜 transport만 사용한다.

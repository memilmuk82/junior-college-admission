# 자료 정책

- 공식 자료 원본과 사용자 업로드는 로컬 전용이며 Git과 Docker 이미지에 넣지 않는다.
- 원본·중간·게시 작업 자료는 각각 `data/raw/`, `data/staging/`, `data/published/`에 둘 수 있으나 모두 버전 관리에서 제외한다.
- 외부 공개가 가능하고 사람 검수를 완료한 정제 전형 데이터만 `data/seed/`에 커밋할 수 있다.
- 자료 상태는 `data/sources/index.yaml`에 기록하며 근거가 부족하면 공개하지 않는다.
- 실제 학생 자료와 그 파생물은 테스트, 로그, 예시 문서에 사용하지 않는다.
